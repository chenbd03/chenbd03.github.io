排序算法-总结

Author: chenbd03 Date: March 11, 2022  21:43:52 Category: 排序算法总结

一、排序基本概念

针对排序，更重要理解排序的思想，时空复杂度，稳定性以及各种排序的适应场景；重点理解快排，归并，堆排；下面排序算法默认排序后均为从小到大

二、插入排序

  • 直接插入排序

    • 核心思想：在序列L[1…n]中，将元素L[i]插入到已有序的子序列中[1…i-1]，其中2≤i≤n，具体做法：

      • 1、找出在L[1…i-1]中的插入位置k
      • 2、将L[k…i-1]的位置全部往后挪动一个位置
      • 3、将元素L[i]插入位置k中

    • 时空分析：时间复杂度O($n^2$)， 空间复杂度O(1)

    • 稳定性：稳定

    • 代码实现：

  • 折半插入排序

    • 核心思想：

      • 1、从前面有序子表L[0…i]中查找出待插入元素的插入位置k–显然是二分查找的操作
      • 2、将有序子表中L[k…i]整体往后移一位，再将元素插入

    • 时空分析:元素的移动次数未改变，依赖于初始状态， 时间复杂度O($n^2$)， 空间复杂度O(1)

    • 稳定性:稳定

    • 代码实现

  • 希尔排序

    • 核心思想：将相隔某个“增量”的记录组成一个子表，对各个子表分别进行直接插入排序，当整个子表元素“基本有序”时，整体再做一次直接插入排序，通过这个增量每次选择总体的一半，最后使得这个增量=1

    • 时空分析：时间复杂度约为O(n^1.3^ )，最坏时间复杂度O($n^2$)， 空间复杂度O(1)

    • 稳定性：不稳定

    • 代码实现：

三、交换排序

  • 冒泡排序

    • 核心思想：从后往前（或从前往后）两两比较相邻元素，若为逆序，则交换，每次冒泡，都能确定一个元素位置；

    • 时空分析：时间复杂度O($n^2$)， 空间复杂度O(1)

    • 稳定性：稳定

    • 代码实现：

  • 快速排序

    • 核心思想：在待排序表L[1..n]中任取一个元素pivot作为基准元素，通过一趟排序（一次划分）后将待排序表划分为独立的两部分L[1…k-1]和L[k+1…n]，使得L[1…k-1]的所有元素小于pivot， L[k+1…n]中的所有元素大于等于pivot；然后分别递归两个子表重复上述工作；

    • 时空分析：平均时间复杂度O(nlogn)，平均空间复杂度O(logn)；最坏情况：时间复杂度O($n^2$)， 空间复杂度O(n)->栈的深度

    • 稳定性：不稳定

四、选择排序

  • 简单选择排序

    • 核心思想：第i趟中，从序列中L[i+1..n]挑选比L[i]还小的元素,同L[i]交换;即每次挑选最小的元素同前面的互换,可见每次选择后都可以确定一个元素位置

    • 时空分析：时间复杂度O($n^2$)， 空间复杂度O(1)

    • 稳定性：不稳定

    • 代码实现

  • 堆排序

    • 核心思想：序列当且仅当满足 L(i) ≥ L(2i) and L(i) ≥ L(2i + 1) 【大根堆】 或者 L(i) ≤ L(2i) and L(i) ≤ L(2i + 1) 【小根堆】， 其中i∈[1, i/2], 实则可将其看成一棵完全二叉树；这里以大根堆为例（小根堆则相反），其特点：大根堆最大元素是存放在根结点的， 即对于完全二叉树来说， 任意一棵子树，其根节点必须≥其儿子结点；针对其排序，其过程大致如下：

      • 1、将无序序列初始化成大根堆

      • 2、将堆顶元素同序列最后一个元素交换，然后将剩下元素重新调整成大根堆；

      • 重复进行，直到堆中剩下一个没被交换的元素为止

    • 时空分析：时间复杂度O(nlogn)，空间复杂度O(1)

    • 稳定性：不稳定

    • 代码实现

        // 数组从1开始, 便于理解完全二叉树中父与子的结点关系
        #include <stdio.h>
        #include <stdlib.h>

        void print_arr(int *arr, int n){
        int i;
        for(i=0;i<n;i++){
        printf(“%d “, arr[i]);
        }
        }

        void swap(int *arr, int i, int j){
        int temp = arr[i];
        arr[i] = arr[j];
        arr[j] = temp;
        }

        void HeapAdjust(int arr[], int k, int n){
        // 数组0位置存放根节点
        arr[0] = arr[k];
        int i;
        for(i=2*k;i<=n;i*=2){
        // k为父节点, i和i+1为其儿子结点
        if (i < n && arr[i] < arr[i+1]){ // 儿子结点进行比较,挑选较大的儿子结点
        i++;
        }
        if (arr[0] >= arr[i]){ //若原本根节点就大,则直接退出
        break;
        }else{ // 否则, i结点跟父结点
        arr[k] = arr[i];
        k = i;
        }
        }
        arr[k] = arr[0];
        }

        void BuildMaxHeap(int arr[], int n){
        int i;
        //由于数组从1开始, 因此2i和2i+1为其儿子结点, 因此对于一颗完全二叉树,i<= n/2, 均为分支结点
        for(i=n/2;i>0;i–){
        HeapAdjust(arr, i ,n);
        }
        }

        void HeapSort(int arr[], int n){
        //初始化大根堆
        BuildMaxHeap(arr, n);
        int i;
        for(i=n;i>1;i–){
        swap(arr, i, 1);
        HeapAdjust(arr, 1, i-1); //将剩余的i-1个元素重新调整
        }
        }

        int main(){
        int n = 6;
        int arr[] = {-1, 5,1,7,4,3};
        HeapSort(arr,n-1);
        print_arr(arr, 6);
        }

  • 基数排序

    • 核心思想：不基于比较和移动进行排序，而基于关键字各位的大小进行排序；
    • 时空分析：时间复杂度：O(d(n+r))， d表示需要进行d趟分配和收集，r为r个队列；空间复杂度O(r)
    • 稳定性：稳定， 与序列的初始状态有关

【补充】计数排序

  • 总体概述：计数排序是一个非基于比较的排序算法。它的优势在于在对一定范围内的整数排序时，它的复杂度为Ο(n+k)（其中k是整数的范围），快于任何比较排序算法。当然这是一种牺牲空间换取时间的做法。时空复杂度均为O(n)，是一种稳定的算法
