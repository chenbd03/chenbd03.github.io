一、网络应用模型

二、网络应用模型

  • C/S(客户/服务器模型)
    • 服务器(提供计算服务的设备)
      永久提供服务
      永久性访问地址/域名
      客户(请求服务计算的主机)
      与服务器通信,使用服务器提供的服务
      间歇性使用网络
      可能使用动态IP地址
      不与其他客户机直接通信
      应用
      Web、文件传输FTP、远程登录、电子邮件等
  • P2P(点对点/对等模型)
    • 特点
      • 不存在永远在线的服务器
      • 每台主机可以提供服务，也可以请求服务
      • 任意端的系统/结点可以直接通讯
      • 节点间歇性接入网络
      • 节点可能改变IP地址
      • 可扩展性好
      • 网络健壮性强

二、应用进程相关

  • DNS（域名系统）

    • 作用：把便于人民记忆的具有特定含义的主机名转换为便于机器处理的IP地址

    • 特点：DNS采用客户/服务器模型,其协议运行在UDP之上,使用53端口号

    • 组成：

      • 层次域名空间：采用层次树状结构,任何连接到因特网的主机或路由器都有一个域名

        • 每个域名都有标号序列组成,而各标号之间用”.”分隔开

        • 域名中标号规定
          英文不区分大小写
          除字符-外不能有其他字符
          每个标号<=63个字符,多标号组成最长域名<=255
          级别最低域名写在最左边,级别最高的顶级域名写在最右边

        • 域名类别分类

          • 顶级域名：.cn .us .uk 等，通用顶级域名：.com .net .org .gov等，基础结构域名(反向域名)：arpa,用于反向解析域名

          • 二级域名
            …

      • 域名服务器

        • 特点：服务器所负责管辖的范围称为区；DNS使用的域名服务器以层次结构组织

      • 解析器（域名解析过程，要求这两种方法都会描述解析过程！）

        • 递归查询

        • 迭代查询（递归与迭代相互查询）

  • 文件传输协议FTP

    • 特点

      • 提供不同种类主机系统之间的文件传输能力
      • 以用户权限管理的方式提供用户对远程FTP服务器上的文件管理能力
      • 以匿名FTP的方式提供公用文件共享的能力
      • 采用客户/服务器的工作方式,使用TCP可靠传输

    • FTP服务器：FTP服务器必须在整个会话期间保留用户的状态信息

      • 主进程：用于接受新的请求
      • 从属进程：负责处理单个请求

    • 工作流程

      • 控制连接(FTP服务器的21端口)
        • FTP客户发出传送请求,通过控制连接发送给服务器端的控制进程
        • 不用来传送文件，在整个会话期间一直保持打开状态
      • 数据连接(FTP服务器的20端口，此端口是可变的，但默认为20)
        • 用来连接客户端和服务器端的数据传送进程
        • 是否使用TCP 20端口建立数据连接与传输模式有关
          • 主动方式(PORT)用TCP20 端口—客户机”主动”
          • 被动方式(PASV)由服务器和客户端自行协商决定—客户机”被动”

    • 传输模式：文本模式和二进制模式

  • 万维网和HTTP

    • 万维网www (采用C/S方式)

      • URL:负责标识万维网上的各种文档，具有唯一标识符URL
        • 一般形式:<协议>://<主机>:<端口>/<路径>
      • HTTP:应用层协议，使用TCP可靠传输
        HTML:文档结构的标记语言

    • HTTP

      • 特点

        • 使用TCP协议,保证数据可靠传输
        • HTTP本身是无连接的,虽然使用TCP,但通信双方在交换HTTP报文时不需要建立HTTP连接
        • 使用Cookie加数据库的方式跟踪客户活动,具有唯一的识别码

      • 连接方式

        • 非持久连接(Close)
          • 每个网页元素对象的传输都需要单独建立一个新的TCP连接,(第三次握手的报文段捎带了客户对万维网文档的请求)
          • 每次TCP连接都需要2个RTT+文档传输的时间
        • 持久连接(Keep-alive)
          • 非流水线(类似停止等待思想)
            客户在收到前一个响应后才能发出下一个请求
            服务器发送完一个对象后,TCP连接处于空闲阶段,浪费服务器资源
          • 流水线(HTTP1.1默认方式)
            可以逐个连续发出各个引用对象的请求
            若所有请求和响应都是连续发送的,那么所有引用的对象共经历一个RTT延迟

      • 报文结构

        • 请求报文

        • 响应报文:每一行结束的地方都有”回车”和”换行”,整个首部运行结束后,还有一空行将首部行和主体分开

        • 【补充】请求方法

  • 电子邮件

    • 组成结构

      • 用户代理
        • 用户与电子邮件交互的接口，比如电子邮件客户端软件
        • 功能：撰写，显示，处理，通信
      • 邮件服务器
        • 功能：发送和接受邮件；向发信人报告邮件传送的情况
          注意：SMTP客户和SMTP服务器
      • 协议
        • 邮件发送协议(SMTP)：用户代理向邮件服务器发送邮件,或者邮件服务器之间
        • 邮件接收协议(POP3、IMAP)：用户代理从邮件服务器读取邮件

    • 相关协议（结合组织结构进行理解）

      • SMTP协议（简单邮件传输协议）

        • 特点：可靠且有效、采用客户/服务器方式、采用TCP连接,端口号25

        • 通信过程

          • 连接建立

          • 邮件传送

          • 连接释放

        • 缺点：只能传输7比特ASCII码内容、无法传送可执行文件及二进制对象

      • POP3、IMAP协议

        • POP：用户代理向邮件服务器发出请求,’拉’取用户邮箱中的邮件
          • 特点：客户/服务器、使用TCP,端口号110
          • 工作方式：下载并保留(邮件服务器) or 下载并删除
        • IMAP：基于POP,另一特性运行用户代理只获取报文的某些部分

      • MIME协议：可支持图片,视频,声音,多种国家语言等,是基于SMTP之上

    • 基于万维网的电子邮件：用户浏览器与Hotmail或Gmail的邮件服务器之间的邮件发送或接收使用HTTP，但不同邮件服务器之间发送邮件就需要用到SMTP
