第四章-网络层

Author: chenbd03 Date: March 10, 2022  10:59:25 Category: 计算机网络

一、网络层功能

向上只提供简单灵活的、无连接的、尽最大努力交付的数据报服务；其功能：异构网络互联、路由与转发、拥塞控制（通信子网中，因出现过量的分组而引起网络性能下降的现象称为拥塞）

二、IP数据报格式

固定部分：

  • 首部长度：单位是4B，占4位，可表示最大十进制数15，可见最大值为4Bx15=60B，但最常见首部长度为20B

  • 总长度：单位是1B，占16位最大长度是2^16-1=65535B

  • 生存时间TTL：占8位,数据报在网络中可通过的路由器数的最大值,没经过一个路由器,TTL减1,直至TTL为0,则该分组放弃

  • 协议：占8位,分组携带的数据使用哪种协议,通常6代表TCP,17代表UDP

  • 首部校验和：占16位,IP数据报的首部校验和只校验分组的首部,而不校验数据部分

  • 标识：每创建一个IP数据报,源主机都会为该数据报加上一个标识号; 路由器将数据报分片时,形成的每个数据报(片)都具有原始数据报的标识号

  • 标志：占3b,后2b有意义

    • DF
      DF=0,该数据报可以分片
      DF=1,该数据报不可以分片

    • MF
      MF=1,表示相应的原始数据报还有后续的片
      MF=0,表示为原始数据报的最后一个片
      可见,只有DF=0,MF才有0和1之分

  • 片偏移：确定片应放在IP数据报的哪个位置，以8B为单位,除最后一个分片,其余分片长度一定是8B的整数倍，例子：可见数据部分1480B+首部20B=MTU(1500B)

三、IPV4地址

  • 分类得IP地址

    • 地址划分：

      主机数A:2^24-2 B:2^16-2 C:2^8-2， 减去全0和全1的地址

    • 特点

      • IP地址分为网络号和主机号,由32比特(4B)的全球唯一标识符
      • 同一局域网上的主机或路由器的IP地址中的网络号必须是一样的
      • IP地址上的一个路由器必须至少有2个IP地址

    • 不能做主机的IP地址

      • 主机号全为0，表示网络本身，
      • 主机号全为1表示本网络的广播地址，又称直接广播地址
      • 127.0.0.1为回环地址
      • 32为全0，即0.0.0.0表示网络上的本主机
      • 32位全1，255.255.255.255表示整个TCP/IP网络的广播地址

    • 特殊的IP地址

  • 网络地址转NAT

    • 作用：通过将专用网络地址转换位公用地址,从而对外隐蔽内部管理IP地址

    • 私有IP地址（私有IP地址不能直接用于Internet,必须通过网关利用NAT把私有IP地址转换为Internet中合法的全球IP地址后才能用于Internet）

      • A类:10.0.0.0~10.255.255.255

      • B类:172.16.0.0~172.31.255.255

      • C类:192.168.0.0~192.168.255.255

      • 注意:目的地址是私有地址,路由器对数据报一律不转发

  • 子网划分与子网掩码

    • 子网划分
      • 将二级IP地址转换为三级IP地址IP地址={网络号,子网号,主机号}
      • 子网划分是单位内部的事情,单位对外仍表现为没有划分子网的资格
      • 从主机号借用若干比特作为子网号,注意,主机号至少有2位
      • 对分类的IPv4地址进行子网划分,子网号不能全0或全1,
      • 无论是分类的IPv4还是CIDR,子网中的主机号为全0或全1的地址不能被指派
    • 子网掩码
      • 默认的子网掩码
        • A类:255.0.0.0
        • B类:255.255.0.0
        • C类:255.255.255.0
      • 无论是二级IP地址还是三级IP地址,子网掩码规定:主机号全0,网络号或子网号全1
      • 外部的IP地址+子网掩码,通过”与”运算,可得到子网的网络号
    • CIDR（无分类编址）
      • 表示法：IP地址/网络前缀所占比特数
      • 特点：
        • CIDR地址掩码相当于子网掩码
        • CIDR消除了对ABC类网络的划分,在软件的支持下实现超网构造的一种IP地址划分
        • CIDR地址块中的地址数一定是2的整数次幂,实际可指派IP地址数为$2^N-2$,N表示主机号的位数
      • 路由聚合(或构成超网)
        • 将网络前缀都相同的连续IP地址进行聚合(通俗讲:若多个网络具有相同的转发接口,则可用路由聚合,多个网络”求交集”)
        • 注意: 划分子网是IP地址少->多;构成超网IP地址多->少
      • 最长前缀匹配：查找路由时匹配的IP地址不止一个,此时应从匹配结果中选择具有最长网络前缀的路由

四、重要协议：ARP、DHCP、ICMP、ICGP与组播路由选择协议

ARP协议（地址解析协议）

  • 目的：完成IP地址到MAC地址的映射；
  • 特点
    • 工作在网络层
    • IP地址到MAC地址的解析是自动的
    • ARP缓存也有生存周期TTL
  • 工作原理
    • 1、 每台主机都会在自己的ARP缓冲区中建立一个 ARP列表，以表示IP地址和MAC地址的对应关系。
    • 2、当源主机发送数据包到目的主机时，首先检查自己 ARP列表中是否存在该 IP地址对应的MAC地址，如果有，就直接将数据包发送到这个MAC地址；如果没有，就使用通过目的MAC地址为FF-FF-FF-FF-FF-FF的帧来封装并广播ARP请求分组
    • 3、网络中所有的主机收到这个ARP请求后，会检查数据包中的目的IP是否和自己的IP地址一致。如果不相同就忽略此数据包；如果相同，该主机首先将发送端的MAC地址和IP地址添加到自己的ARP列表中，如果ARP表中已经存在该IP的信息，则将其覆盖，然后给源主机发送一个ARP响应数据包，告诉对方自己是它需要查找的MAC地址；
    • 4、源主机收到这个ARP响应数据包后，将得到的目的主机的IP地址和MAC地址添加到自己的ARP列表中，并利用此信息开始数据的传输。
    • 5、如果源主机一直没有收到ARP响应数据包，表示ARP查询失败。
  • APR的4种典型情况
    • 发送方是主机
      同一个网络的主机，用ARP找到目的主机的MAC地址
      不同网络的主机，ARP找到本网络上的一个路由器MAC地址，剩下工作交给它
    • 发送方是路由器
      同一个网络主机。ARP找到目的主机的MAC地址
      不同网络的主机，ARP找到本网络上的一个路由器MAC地址，剩下工作交给它

DHCP协议（动态主机配置协议）

  • 目的：常用于给主机动态分配IP地址

  • 特点

    • DHCP是应用层协议，基于UDP
    • 使用客户/服务器方式工作
    • 客户端/服务器端需要广播方式交互
    • DHCP服务器分配给DHCP客户机的IP地址是临时的

ICMP协议（网际控制报文协议）

  • 目的：提高IP数据报交付成功机会,通过使用ICMP协议向主机或路由器报告差错和异常情况
  • 特点：
    • ICMP是IP层(网络层)协议
    • ICMP作为IP数据报的数据,加上数据首部组成IP数据报发送出去
  • 种类
    • ICMP差错报文
      • 需要向源主机报告ICMP差错报文
        • 终点不可达：数据无法交付
        • 源点抑制：拥塞丢数据
        • 时间超过：TTL=0
        • 参数问题：首部字段有问题
        • 改变路由(重定向)：有更好的路由
      • 不需要向源主机发送ICMP差错报文
        • 对ICMP差错报文本身
        • 具有组播地址的数据报
        • 对第一个分片的数据报片的后续数据报片
        • 对特殊地址,比如127.0.0.1或0.0.0.0
    • ICMP询问报文
      • 回送请求和回答报文：分组间探测PING,其工作在应用层
      • 时间戳请求和回答报文：Traceroute ,其工作在网络层
      • 掩码地址请求和回答报文
      • 路由器询问和通告报文

ICGP协议 与 组播路由选择协议（IP组播）

​ IP组播

  • 相关概念

    • 组播机制：让源计算机一次发送的单个分组可抵达用一个组地址标识的若干目标主机，并被它们正确接收
    • 组播（多播）需要路由器才能实现，主机组播时只需发送一次数据即可，可减少网络负载和发送者负担
    • IP组播运用到了ICGP协议

  • IP组播地址

    • D类地址，前四位是1110，范围是：224.0.0.0~239.255.255.255，每个D类IP地址标志一个组播组

    • 特点

      • 不可靠交付
      • 只能用于目的地址，不可用于源地址
      • 对组播数据报不产生ICMP数据报
      • 并非所有D类地址都做组播地址

    • D类IP地址与以太网组播地址的映射：组播MAC地址以16进制值01-00-5E打头，余下6个16进制位是根据IP组播地址的后23位转化而来的

ICGP协议 与 组播路由选择协议

  • 使路由器知道组播成员的信息，需要用到IGMP协议
  • 组播路由选择协议组播转发树，可以避免路由环路

五、路由算法与路由选择协议

  • 路由算法
    • 静态路由算法：手工配置路由信息
    • 动态路由算法
      • 距离向量路由算法，例如RIP
      • 链路状态路由算法，例如OSPF
  • 路由选择协议

    • 内部网关协议

      • RIP 路由信息协议：一种分布式的基于距离向量的路由选择协议

        • 规定
          • 距离向量：从自身到其他每个目的网络的唯一最佳距离记录
          • 跳数：每经过一个路由器，距离（跳数）+1
          • 每条路径最多只包含15个路由器，当距离=16，表示网络不可达；只适用小型网络
          • 不支持子网掩码的RIP广播，必须保证RIP中每个网络的子网掩码相同
        • 特点
          • 仅和相邻路由器交换信息
          • 交换的信息是自己的路由表
          • 按固定的时间间隔交换信息，例如规定的30s
          • RIP是应用层协议，使用UDP传送数据
          • 缺点：网络故障，“坏消息传得慢”，使收敛的时间变长，【注意】收敛：指路由环境发送变化后，各路由器调整自己的路由表以适应网络拓扑结构的变化，最终达到稳定状态
        • 距离向量算法
          • 1、对地址为 X 的相邻路由器发来的RIP报文，先修改此报文的所有项目：把“下一跳”字段中的地址都改为 X ，并把所有的“距离”字段值加 1
          • 2、对修改后的RIP报文中的每个项目，执行以下步骤
            • 若原来的路由表中没有目的网络N，把该项目添加到路由表中
            • 若原来的路由表中有目的网络N，且下一跳路由器地址是 X，则更新路由表表项
            • 若路由表中有目的网络N，但下一跳地址不是X，若收到的项目中的距离 d 小于路由表中的距离，则进行更新，否则什么也不做

      • OSPF 开放最短路径优先：使用分布式链路状态路由算法

        • 特点

          • 向本自治系统中的所有路由器发送信息，使用洪泛法
          • 发送的信息时与本路由器相邻的所有路由器的链路状态，包括和哪些路由器相邻、该链路的度量/代价Metric
          • 只有当链路状态发送变化时，路由器才用洪泛法向所有路由器发送此信息
          • OSPF是网络层协议，不使用UDP或TCP，直接用IP数据报传送，收敛快
          • 使用Dijkstra最短路径算法计算从自己到各个网络的最优路径

        • 五种分组类型

          • Hello问候分组：用来发现和维持邻站可达性

          • 数据库描述分组：摘要信息

          • 链路状态请求分组LSR：向对方请求发送某些链路状态项目的详细信息

          • 链路状态更新分组LSU：洪泛法对全网更新链路状态

          • 链路状态确认分组LSAck：对链路更新分组确认

    • 外部网关协议

      • BGP 边界网关协议：BGP力求寻找一条能够达到目的网络且比较好的路由，而并非是最好路由
        • 特点
          • 采用路径向量路由选择
          • BGP是应用层协议，基于TCP
          • BGP支持CIDR
          • BGP路由器发送变化时，只更新有变化的部分
        • BGP-4类型：
          • 打开（open）报文：用来相邻的另一个BGP发言人建立关系
          • 更新（update）报文：用来发送某一路由信息，以及列出要撤销的多条路由
          • 保活（Keeplive）报文：用来确定打开报文并周期性地证实邻站关系
          • 通知（Notification）报文：用来发送检测到的差错
        • 原理：每个自治系统的管理员至少选择一个路由器作为自治系统的“BGP发言人”，一个发言人要和其他自治系统的BGP发言人交换信息，必须先建立TCP连接，在此连接基础上交换BGP报文以建立BGP会话，再利用BGP会话交换路由信息

      RIP、OSPF、BGP三者比较

六、IPV6 和 移动IP

  • IPV6
    • 地址表示形式
      • 每4位用一个十六进制数表示,并用冒号分隔每16位
      • 相继的0值域时,可用双冒号::表示,在一个地址中仅能出现一次
    • IPV6与IPv4对比
      • IPv6将地址从IPv4的32位地址增大到128位地址,字节数16B是IPv4字节数4B的平方
      • IPv6将IPv4的校验和字段移除,以减少每跳的处理时间
      • IPv6首部长度是8B的整数倍,而IPv4是4B的整数倍
      • IPv6不允许分片,而IPv4可在路由器和主机允许分片
      • IPv6支持即插即用
    • IPV6向IPv4过渡
      • 双协议栈：一台设备上同时装有IPv4和IPv6协议栈
      • 隧道技术：整个IPv6数据报封装到IPv4数据报的数据部分
  • 移动IP
    • 移动IP技术是指移动结点以固定的网络IP地址实现跨越不同网段的漫游功能,并保证基于网络IP的网络权限在漫游过程中不发生任何改变
    • 移动IP为移动主机设置两个IP地址:主地址和辅地址; 主地址固定,辅地址动态改变

七、 网络层设备

  • 路由器体系结构

    • 路由选择部分

      • 路由选择处理机：根据特定的路由选择协议构造出路由表，同时经常定期和相邻路由器交换路由信息而不断更新和维护路由表信息
      • 路由选择协议
      • 路由表

    • 分组转发部分

      • 交换结构：根据转发表对分组处理，将某个输入端口进入的分组从一个合适的输出端口转发出去

      • 一组输入端口

      • 一组输出端口

  • 路由器功能：

    • 路由选择：发生在路由器之间
      • 根据特定的路由选择协议构造出路由表，同时经常定期和相邻路由器交换路由信息而不断更新和维护路由表信息；
      • 【注意】路由表：由路由算法得来的，通常包括：目的网络IP地址、子网掩码、下一跳IP地址、接口，通常由软件实现
    • 分组转发：发生在路由器内部
      • 路由器根据转发表将用户的IP数据报从合适的端口转发出去，【注意】转发表包含发往的目的地址、分组的下一跳（实际MAC地址）

  • IP分组的交付方式

    • 直接交付：数据报从源主机到目的主机是同一个网络上,则直接交付不需要路由器
    • 间接交付：数据报从源主机到目的主机是不同一个网络上,则路由器通过路由表指出的路由将数据报发送到下一个路由器,称为间接交付

  • 存储转发设备之间的互联

    • 路由器：可互联两个不同网络层协议的网段
    • 网桥，交换机：可互联物理层和数据链路层两个不同协议的网段
    • 集线器、中继器：不能互联物理层两个不同协议的网段
