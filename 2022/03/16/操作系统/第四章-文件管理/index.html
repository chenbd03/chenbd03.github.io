第四章-文件管理

Author: chenbd03 Date: March 16, 2022  10:16:14 Category: 操作系统

一、文件系统基础（只需了解即可！）

1、文件逻辑结构

  • 无结构文件（流式文件）：文件内部的数据其实就是一系列字符流，没有明显的结构特性。比如txt文件等

  • 有结构文件（记录式文件）

    • 顺序文件：文件中的记录一个接一个地顺序排列（逻辑上），记录可以是定长的或可变长的。各个记录在物理上可以顺序存储或链式存储

    • 索引文件

    • 索引顺序文件

2、文件目录结构(重点理解FCB和索引结点)

  • 文件控制块和索引结点

    • 文件控制块FCB：一个文件对应一个FCB，一个FCB就是一个目录项，多个FCB组成文件目录

    • 索引结点（FCB的改进）：除了文件名之外的所有信息都放到索引结点中，每个文件对应一个索引结点

      • 目录项只包含文件名、索引结点指针，因此每个目录项的长度大幅减少

      • 由于目录项长度减小，因此每个磁盘块可以存放更多的目录项，因此检索文件的磁盘IO次数就会减少很多！

  • 目录结构的增删改查以及“树型”的目录结构（可联想到windows上对目录的操作），需要了解当前路径、相对路径和绝对路径相关概念！

3、文件共享

  • 基于索引结点的共享方式（硬链接）
    • 索引结点中设置一个链接计数变量 count，用于表示链接到本索引结点上的用户目录项数，count=x，表明有x个用户目录项链接到该索引结点上（或者说是共享该文件），当count=0时，系统负责删除文件；
    • 若某个用户决定“删除”该文件，则只是要把用户目录中与该文件对应的目录项删除，且索引结点的count值减 1
  • 基于符号链的共享方式（软链接）
    • 在一个Link型的文件中记录共享文件的存放路径（类似于Windows上创建的快捷方式）
    • 注意：即使软链接指向的共享文件被删除，Link型文件仍然存在，只不过Link型文件启动程序打开文件时会失败！（找不到对应目录项）

二、文件系统实现（核心重点！！！）

  • 文件系统的层次结构

  • 文件分配的三种方式（此处可参考王道强化班-整个文件系统示意图！）

    • 连续分配：要求每个文件在磁盘上占有一组连续的块

      • 优点：支持顺序访问和直接访问（即随机访问）；连续分配的文件在顺序访问时速度最快
      • 缺点：不方便文件拓展；存储空间利用率低，会产生磁盘碎片，需要用紧凑技术来处理碎片
      • 【注意】对于计算问题，要把握清楚连续分配时访问某个文件需要访问磁盘次数！以及对应的FCB需要哪些属性！

    • 链接分配

      • 隐式链接：除文件的最后一个盘块之外，每个盘块中都存有指向下一个盘块的指针。文件目录包括文件第一块的指针和最后一块的指针

        • 优点：很方便文件拓展，不会有碎片问题，外存利用率高。
        • 缺点：只支持顺序访问，不支持随机访问，查找效率低，指向下一个盘块的指针也需要耗费少量的存储空间。
        • 【注意】对于计算问题，要把握清楚连续分配时访问某个文件需要访问磁盘次数！以及对应的FCB需要哪些属性！还有每个块中包含哪些？

      • 显式链接：把用于链接文件各物理块的指针显式地存放在一张表中。即 文件分配表（FAT，File Allocation Table）

        • 优点：很方便文件拓展，不会有碎片问题，外存利用率高，并且支持随机访问。相比于隐式链接 来说，地址转换时不需要访问磁盘，因此文件的访问效率更高。
        • 缺点：文件分配表的需要占用一定的存储空间。
        • 【注意】一个磁盘只会建立一张文件分配表。开机时文件分配表放入内存，并常驻内存。

        【注意】考试题目中遇到未指明隐式/显式的“链接分配”，默认指的是隐式链接的链接分配

    • 索引分配：允许文件离散地分配在各个磁盘块中，系统会为每个文件建立一张索引表，索引表中记录了文件的各个逻辑块对应的物理块；索引表存放的磁盘块称为索引块。文件数据存放的磁盘块称为数据块

      • 问题引入：索引块太小，无法支持大文件，因此需要采用下列机制（文字难以描述，可用图来进行辅助理解）：

        • 链接方案

        • 多层索引

        • 混合索引

        【总结】

    三种方式优缺点对比

    【注意】可参考个人做题笔记！核心要点如下：

    1、一个FCB实则就是一个文件目录项！！！注意：在连续分配中，FCB一定要包含：文件名、文件初始块号，文件长度；在链接分配中，FCB包含：文件名、文件起始块、文件结束块；在索引分配中，FCB可包含文件名、inode指针，而inode结点存放文件的相关属性、直接索引、一级间接、二级间接、三级间接等，且索引分配中，操作系统会给每个文件分配一个inode结点

    2、FAT为文件分配表，存放盘块号（隐含）和下一块磁盘块的索引；整个磁盘中仅设置一张，在开机时系统会将它载入内存，且FAT支持随机访问；

    3、对于最大限度减少读磁盘的次数，可采取的方法：

    ①：将当前文件所处的目录设置为“当前目录”，则访问该文件时直接访问当前目录，不需要读磁盘，接着再读入相应的块即可。

    ②：FCB分解法，可参考2022王道书-操作系统P245页第2题。

    ③将当前文件挂到根目录最前面的目录项，则访问该文件时直接访问根目录，不需要读磁盘，接着再读入相应的块即可。

    4、会计算读某个文件最少和最多访问磁盘的次数！即会重点分析查目录的过程！读某个文件以连续分配、链接分配或索引分配求读磁盘次数？至少？至多？参考王道书P248的第14题！

    5、对于一个文件实际所占的空间大小，不仅包括自身文件大小（数据块所占大小），而且还需包括索引块所占大小和索引结点所占大小！！！

    6、对于磁盘块大小（等价说法：块长），磁盘块的每个地址项长度占多少B（等价说法：磁盘地址占多少B、块号长占多少B，链接指针大小为多少B）

    7、文件系统总空间所占的磁盘块数 <=> 磁盘块的每个地址项长度(或链接指针)应占多少bit，例如文件系统空间占磁盘块数为2^32 <=> 反映出磁盘块的每个地址项长度应占32bit=4B.

    8、在文件系统空间中，磁盘块的移动要先读入内存，再进行移动，故在磁盘空间中每次移动一个磁盘块需访问磁盘两次。

    9、对于链接索分配中，块大小=数据大小+链接指针大小；故求系统文件能支持文件的最大长度，注意要将磁盘块大小减去链接指针大小才是实际数据存放的大小。

    10、簇大小（windows说法）=块大小（Linux or UNIX说法）

    11、操作系统给文件分配大小是以块（簇）为单位的，例如：一个文件大于一个块而小于两个块，操作系统还是分两个块给这个文件。

    相关计算公式：

    1、FCB占磁盘多少块=FCB大小xFCB个数 / 磁盘块大小

    2、查找目录文件某个FCB的平均访问磁盘次数：①先计算出FCB占多少磁盘块，假设为n ；②平均访问磁盘次数：(1+2+…+n)xn / 2

    3、硬盘总块数M=硬盘大小/磁盘块大小 => 其FAT表需满足M≤2^n => FAT表中可用nbit表示块号 <=> FAT的每个表目(表项)需要占nbit => 整个FAT表大小=(nbit/8)B * 表项的数量M

    注意:若题目只给了FAT表的每个表项占2B => FAT表中可用(2*8)bit表示块号,即FAT可记录的块号范围为0~2^16-1，故FAT的最大长度为2^16 * 2B = 128KB => 系统支持文件最大长度=2^16 * 磁盘块大小

    4、（混合索引）求系统文件能支持的最大长度文件：

    ①确定索引结点直接索引x、一级间接y、二级间接w、三级间接z占多少块

    ②确定磁盘块大小p，和每个磁盘块能容纳多少个地址项q（要知道地址项长度，或地址项所占空间，且要注意地址项所能表示的范围）。

    ③最大长度文件：（x+yq+wqq+zqqq）* p 即可

    • 空闲表法:属于连续分配方式，系统为每个文件分配一块连续的存储空间，系统为外存上的所有空闲区建立一张空闲盘块表，每个空闲区对应于一个空闲表项。分配和回收同内存类似。

    • 空闲链表法

    • 位示图法:二进制的一位用来表示磁盘中一个盘块的适用情况，通常0表示空闲，1表示已分配

      • 盘块分配,位示图的第i行第j列转化为对应的盘块号;其中n表示每行的位数：

        从1开始：b=n(i-1)+j

        从0开始：b=ni+j

        并将map[i,j]=1

      • 盘块回收,将回收的盘块号转化为位示图的第i行和第j列:

        从1开始:

        i = (b-1)/n+1 ; j = (b-1)%n+1

        从0开始:

        i = b/n; j=b%n

    • 成组链接法

三、磁盘管理

磁盘的相关概念：磁盘、磁道、扇区、如何在磁盘读写数据、盘面、柱面、了解和如何计算磁盘物理地址（柱面号，盘面号，扇区号）

一次磁盘I/O操作需要的时间：寻道时间、延迟时间、传输时间

  • 寻道时间：(现在的硬盘移动一个磁道大约需要0.2ms，磁臂启动时间约为2ms)
  • 延迟时间：通过旋转磁盘，使磁头定位到目标扇区所需要的时间。设磁盘转速为 r （单位：转/秒，或 转/分），则平均所需的延迟时间 TR = (1/2)*(1/r) = 1/(2r)
  • 传输时间：从磁盘读出或向磁盘写入数据所经历的时间，假设磁盘转速为 r，此次读/写的字节数为 b，每个磁道上的字节数为 N。则：传输时间Tt = (1/r) * (b/N) = b/(rN)
  • 总的平均存取时间 Ta = TS + 1/2r + b/(rN)
  • 【补充】

磁盘调度算法（具体计算方法参考王道书-操作系统的课后习题！）

思想 是否会饥饿 优点 缺点
先来先服务FCFS 先后顺序调度 公平简单 平均寻道距离大
最短寻找时间优先SSTF 选择调度处理的磁道是与当前磁头所在磁道最近的磁道 是，磁头在一个区域内来回移动 性能比FCFS好 可能出现饥饿现象，
扫描算法SCAN（其缺点1可通过LOOK调度解决） 只有磁头移动到最外侧磁道时才能往内移动 否 寻道性能较好，不会造成饥饿 1、只有到达边上的磁盘才改变磁头移动方向；2、对各个磁道的响应频率不均
循环扫描C-SCAN（其缺点可通过C-LOOK调度解决） 只有磁头移动到最外侧磁道才改变磁头移动方向，返回时直接快速移动到起始端而不做任何处理 否 消除了两端磁道请求的不公平 只有到达边上的磁盘才改变磁头移动方向

【注意】对于LOOK调度和C-LOOK调度，磁道移动到只需要到达最远端的一个请求即可返回，不需要到达磁盘端点

磁盘的管理

  • 磁盘初始化
    1. 进行低级格式化（物理格式化），将各个此道划分为扇区
    2. 将磁盘分区，每个分区由若干柱面组成
    3. 进行逻辑格式化，创建文件系统，包括创建文件系统的根目录，初始化存储空间管理所用的数据结构
  • 引导块
    • 计算机开机需要一系列的初始化工作，是需要通过执行初始化程序（自举程序）完成的；完整的自举程序放在磁盘的启动块（即引导块/启动分区）上，启动块位于磁盘的固定位置。
  • 坏块的管理
    • 对于简单的磁盘，可以在逻辑格式化时（建立文件系统时）对整个磁盘进行坏块检查，标明哪些扇区是坏扇区，
    • 复杂的磁盘,磁盘控制器（磁盘设备内部的一个硬件部件）会维护一个坏块链表
